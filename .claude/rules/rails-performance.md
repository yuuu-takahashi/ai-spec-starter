# パフォーマンス規約

## データベース

- N+1 クエリを発生させない。関連データは `includes` / `preload` / `eager_load` で事前読み込み
- 集計は Ruby ではなく DB で行う（`count`, `sum`, `group`）
- 必要なカラムのみ `select` / `pluck` で取得。BLOB/TEXT は不要時に除外
- 大量レコードは `find_each(batch_size: 1000)` でバッチ処理

## メモリ

- 大量データを一括で配列に変換しない（`to_a` の暗黙実行に注意）
- 大きなファイルはストリーミング処理する（一括読み込みしない）
- 不要なオブジェクト生成を避ける。ループ内での文字列連結は `<<` か `join`

## 非同期処理

- 重い処理（メール送信、外部 API、画像処理等）は `ActiveJob` で非同期化する
- ジョブは冪等に設計する（リトライで副作用が重複しない）
- キューの優先度を適切に設定する（`default`, `mailers`, `low` 等）

## キャッシュ

- 頻繁にアクセスされる集計結果はキャッシュする
- キャッシュキーにはバージョニング（`cache_key_with_version`）を含める
- キャッシュ無効化のタイミングを明確にする

## DO NOT

- ビューで N+1 を発生させるクエリを実行しない（コントローラで事前ロード）
- `map` / `select` で AR リレーションを Ruby 配列に変換してフィルタしない（DB で絞る）
- ループ内で個別の DB クエリや API 呼び出しを行わない
