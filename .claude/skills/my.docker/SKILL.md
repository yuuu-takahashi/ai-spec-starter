# my:docker

## 概要

ローカルの Docker 環境の使用状況を確認し、無駄なリソースをクリーンアップするスキルです。

## いつ使うか

- Docker のディスク使用状況を確認したい
- 不要なコンテナ・イメージ・ボリュームを削除したい
- ストレージを圧迫している Docker リソースを特定したい
- Docker 環境をクリーンな状態に保ちたい

## 使用方法

```text
/my:docker
```

引数なし。実行すると現在の使用状況を確認し、削除可能なリソースを提案します。

## 実行フロー

1. **現在の使用状況を表示**
   - `docker system df -v` を実行
   - 各カテゴリ（コンテナ、イメージ、ボリューム、キャッシュ）の使用量を表示

2. **詳細なリソース一覧を表示**
   - 実行中のコンテナ
   - 停止中のコンテナ
   - 使用中・未使用のボリューム
   - 未使用のイメージ
   - 未使用のネットワーク
   - ビルドキャッシュ

3. **クリーンアップ提案**
   - 削除可能なリソースを各カテゴリごとに提示
   - 削除可能な容量を表示

4. **ユーザー承認の取得**
   - **重要**: 削除コマンドを実行する前に、必ずユーザーに明示的な承認を求める
   - 削除対象のリソースと容量を明確に表示

5. **クリーンアップ実行**
   - ユーザーの明示的な承認を得た場合のみ実行
   - 安全な順序で実行

6. **結果の確認**
   - クリーンアップ後の使用状況を表示

## クリーンアップオプション

### オプションA: 停止中のコンテナを削除

ユーザーに確認してから実行。

### オプションB: 未使用のボリュームを削除

**注意**: ボリュームを削除するとデータが失われます。

### オプションC: 未使用のイメージを削除

タグのないイメージやダングリングイメージを削除します。

### オプションD: 未使用のネットワークを削除

使用されていないカスタムネットワークを削除します。

### オプションE: ビルドキャッシュを削除

ビルドキャッシュを削除します。次回のビルドが遅くなる可能性があります。

### オプションF: すべてを一度にクリーンアップ

**警告**: 停止中のコンテナ、使用されていないネットワーク、未使用のイメージ、ボリューム、ビルドキャッシュをすべて削除します。

### オプションG: 実行中のコンテナも含めてすべてのコンテナを削除

**警告**: 実行中のコンテナも停止して削除します。実行中のサービスが停止します。

### オプションH: すべてのボリュームを削除（使用中も含む）

**警告**: 使用中のボリュームも削除します。すべてのデータが失われます。

### オプションI: 完全クリーンアップ

**重大な警告**: Docker 環境が完全にクリーンな状態になります。すべてのデータが失われます。

## 注意事項

- **実行中のコンテナの削除**: サービスが停止します
- **ボリュームの削除**: 重要なデータが含まれている可能性があります
- **イメージの削除**: 使用中のイメージを削除するとコンテナの再起動に影響します
- **ネットワークの削除**: コンテナの接続に影響する可能性があります
- **完全クリーンアップ**: すべてのデータが失われます。この操作は取り消せません

## 削除実行のルール

- ユーザーが「いいえ」「キャンセル」「やめる」など拒否した場合、削除を実行しない
- ユーザーが曖昧な返答をした場合、明確な承認を求める
- ユーザーが「はい」「削除して」「OK」など明示的に承認した場合のみ削除を実行

詳細な処理手順は `references/cleanup-steps.md` を参照。
