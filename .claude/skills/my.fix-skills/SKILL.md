# 概要

PostToolUseFailure hook が自動記録するエラーログを分析し、原因となるスキルを特定して**スキル自体を直接修正**する。根本原因のあるスキルの SKILL.md を改善することを主眼とする。

## いつ使うか

- `/my:fix-skills` を実行したとき
- 「スキルを修正して」「エラーログを確認して」と依頼されたとき

## 手順

### 1. エラーログを読み込む

`.claude/error-log.jsonl` を Read ツールで読む。

- ファイルが存在しない場合: 「エラーログファイルがまだ作成されていません。ツール実行でエラーが発生すると自動記録されます」と報告して終了
- ファイルが空の場合: 「記録されたエラーはありません」と報告して終了

各エントリのフォーマット:

```json
{"id":"a1b2c3d4","timestamp":"2026-03-29T03:34:29Z","tool":"Bash","error":"...","status":"open"}
```

`status: "open"` のエントリのみを分析対象とする。

### 2. エラーパターンを分析し、原因スキルを特定する

1. `status: "open"` のエントリだけを抽出する
2. tool 名 + error メッセージのキーで集約し、発生回数を数える
3. 発生回数が多い順にソートする
4. エラーの文脈（タイムスタンプ、ツール名、エラー内容）から**どのスキルの実行中に発生したか**を推定する
5. 以下は除外してカウントだけ報告する:
   - 一時的なエラー（ネットワーク、タイムアウト、DNS）
   - ユーザーが意図的に deny した操作（permission denied 系）

### 3. 修正方法を提案する

各エラーパターンについて、**複数の修正方法をおすすめ度付きで提案**する。以下の選択肢から該当するものを選び、それぞれメリット・デメリットを添える。

| 修正方法 | おすすめ度の目安 | 条件 |
| --- | --- | --- |
| **対象スキルの SKILL.md を直接修正** | 高い | スキルの手順・前提条件・バリデーションに不備がある |
| **CLAUDE.md にルールを追加** | 高い | プロジェクト全体の規約として定着させるべき |
| **hook を追加・修正** | 中程度 | 物理的にブロックすべき操作（書き込み禁止等） |

提案フォーマット（エラーパターンごとに出力）:

```text
## エラー: [短いタイトル]
- 発生: [ツール名] / [回数]回 / 推定スキル: [スキル名]
- 原因: [なぜこのエラーが起きるか]

### 修正方法
1. [おすすめ] 対象スキルの修正 — [具体的な変更内容]
   - メリット: [根本解決できる理由]
   - デメリット: [影響範囲・副作用]
2. ...
```

### 4. ユーザーの選択を受けて修正を実行する

- ユーザーが選んだ修正方法を実行する
- 対象スキルの SKILL.md を修正する場合: Read → 該当箇所を特定 → Edit で修正
- CLAUDE.md への追記や hook 変更はユーザー確認後に実行

### 5. 対応済みエントリのステータスを更新する

修正を実行したエントリの `status` を `"resolved"` に変更する。error-log.jsonl を Read → 該当エントリの status を書き換え → Write で上書き保存する。

解決済みエントリは**削除しない**（履歴として残す）。ログが肥大化した場合のみ、ユーザーに確認の上で resolved エントリを削除する。

### 6. 報告する

以下をユーザーに報告する:

- エラーパターンごとの修正方法の提案（おすすめ付き）
- ユーザー選択後に実施した修正内容のサマリー
- 一時的なエラーとして除外したもの（件数のみ）

## トラブルシュート

| 問題 | 原因 | 対処 |
| --- | --- | --- |
| error-log.jsonl が生成されない | PostToolUseFailure hook が未設定 | `.claude/settings.json` の hooks に PostToolUseFailure を追加する |
| JSONL のパースに失敗する | log-error.sh の jq 出力が壊れている | `.claude/hooks/log-error.sh` を確認し、jq コマンドのエラーがないか検証する |
| 原因スキルが特定できない | エラーの文脈情報が不足 | ユーザーに「どのスキル実行中に発生したか」を確認する |
