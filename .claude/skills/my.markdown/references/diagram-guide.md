# Mermaid 図ガイド

Mermaid には複数の図の型があり、状況に応じて最適なものを選ぶことで可読性が上がる。以下は代表的な型と使いどころです。

## 1. Flowchart (graph TD/graph LR)

- **用途**: 手順・意思決定・オペレーションフローを示すとき。
- **特徴**: 箇条書きのステップをノード化し、矢印で遷移を示す。条件分岐は `||`、ループは `-->` で表現。
- **例**: オンボーディング手順・申請ワークフロー・レスポンス判断。

## 2. Sequence diagram

- **用途**: システム間のメッセージやタイミングを示すとき。
- **特徴**: アクターとシステム間の呼び出し順序を時間軸で表現。非同期/同期、戻り値なども記述可能。
- **例**: API 呼び出しのログインシーケンス・チャット RPC のやり取り。

## 3. Gantt chart

- **用途**: スケジュール・プロジェクトマイルストーンを示すとき。
- **特徴**: 期間・依存をバーで記述し、マイルストーンを強調。`dateFormat` や `section` で構成。
- **例**: リリースプラン・ロードマップ・スプリント計画。

## 4. Class diagram / Entity Relationship

- **用途**: データモデル・オブジェクト構造を説明する。
- **特徴**: クラス（ボックス）に属性・メソッドを記述し、関連（`-->`, `--|>`, `o--`）を示す。
- **例**: DB スキーマ、API DTO、ドメインオブジェクト間の関係。

## 5. State diagram

- **用途**: 状態遷移・ライフサイクルを表すとき。
- **特徴**: `state` ブロックで状態を定義し、条件付きの遷移矢印を追加。entry/exit も記述できる。
- **例**: ドキュメントレビューのステータス、CI パイプラインの段階。

## 6. Journey / Timeline

- **用途**: ユーザー旅路や時系列イベント。
- **特徴**: `timeline` または `journey` を使い、時間軸とイベントを並べる。テキストを使った注釈も可能。
- **例**: ユーザーオンボーディング、トラブルシュートの時間経過。

## 7. Pie / Quadrant / Radar

- **用途**: 比率・分類・バランス感を視覚化したいとき。
- **特徴**: `pie` で割合、`quadrantChart` で軸の位置、`radarChart` で多軸比較を表す。
- **例**: KPI バランス、比較チャート、品質指標の分布。

## 8. Mindmap / Graph with subgraphs

- **用途**: 概念整理・ツリー構造・機能一覧。
- **特徴**: `graph TD` でも `subgraph` を使って階層を表現し、中央から放射状に展開できる。
- **例**: 機能一覧、製品マップ、調査結果の分類。

## 補足

- 図の目的を書いた上で、`theme`（default/dark）や `classDef` で色を統一し、`%%` コメントで注釈を入れる。
- Mermaid MCP で生成する際には `outputType: "file"` を使えば PNG/SVG をファイルとして取得できるので、Markdown では `![Caption](assets/diagram.svg)` で挿入する。
- 図を使うべきタイミングを迷うときは、`MySkillCreator` で「この説明はステップ/状態/時系列どれに近いか？」を質問すると、最適な型を推薦してくれる。
