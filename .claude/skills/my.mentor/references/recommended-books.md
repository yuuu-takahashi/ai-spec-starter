# 推薦図書ガイド

## 選書基準

以下の **すべて** を満たすものを掲載する。追加・入れ替え時もこの基準で判断する。

| # | 基準 | 理由 |
| --- | ------ | ------ |
| 1 | **Amazon 評価 ≥ 4.5**（レビュー 50件以上） | 定量的な品質フィルタ。少数レビューの高評価は除外 |
| 2 | **5年以上読まれている、または版を重ねている** | フレームワーク本は陳腐化が速い。思考法・設計・判断力に関わる本は長く使える |
| 3 | **ジュニア〜ミッドが主対象** | メンタースキルのターゲット層。上級者向けは「余力があるとき」ラベル付き |
| 4 | **日本語で現在も入手可能**（翻訳 or 日本語オリジナル） | 学習の初速を落とさない |
| 5 | **既存の本と層が重複しない** | 1層1冊に絞り、選択疲れを防ぐ |

## 初心者向け: 最初の一歩ガイド

初心者（Lv1〜Lv2: 用語が曖昧・何がわからないかわからない）には、**いきなり本を渡さない**。まず全体像を掴むメディアから入り、手を動かし、必要になったら本へ進む。

### レベル別おすすめ順序

```text
全体像を知る（動画・地図）→ 手を動かす（ハンズオン）→ 体系的に読む（本）
```

| ステップ | メディア | 具体例 | 狙い |
| ---------- | ---------- | -------- | ------ |
| 1. 全体像 | 動画・ロードマップ | YouTube の概要動画（10〜20分）、[roadmap.sh](https://roadmap.sh/) | 「何の話か」「どこに向かうか」の地図を手に入れる |
| 2. ハンズオン | 対話型教材 | [Progate](https://prog-8.com/)、[ドットインストール](https://dotinstall.com/) | 手を動かして「動いた」体験を得る。完璧に理解しなくてよい |
| 3. 体系化 | 動画コース or 本 | [Udemy](https://www.udemy.com/)（目次・レビュー確認）、下記の推薦図書 | ハンズオンで生まれた疑問を体系的に埋める |

### 提案時の声かけ例

- **全体像が見えていない場合:** 「まず ○○ の概要動画を 1 本見て、何の話かザッと掴んでから手を動かすと迷いが減りますよ」
- **手を動かしていない場合:** 「Progate/ドットインストールで ○○ のコースを 1 周するだけでも、用語のイメージが変わります」
- **ハンズオンは済んで疑問がある場合:** 「その疑問なら『○○』の △△ 章がちょうどいいです」← ここで初めて本を提案

### YouTube 概要動画の選び方

特定チャンネルは推さない。以下の基準で選ぶ：

- **10〜30分で全体像をカバー**（長すぎると完走しない）
- **図やアニメーションが多い**（テキスト読み上げだけのものは避ける）
- **公開から 2年以内**（技術トピックは鮮度が重要）
- **日本語 or 英語**（ユーザーの得意な方で。英語なら字幕ありを確認）

## 提案時の注意

- **1回に提案するのは 1〜2冊まで。** 選択肢が多いと読まれない
- 提案時は **「なぜこの本か」を詰まりの分類と紐づけて** 1行で伝える
- Amazon のレビュー評価・版・刊行年は変動する。提案前に最新情報を確認できるなら望ましい
- アルゴリズム専門書は版と難易度が幅広いので、**必要になったトピック**（グラフ、動的計画法など）が決まってから、薄い入門 → 必要なら大学テキスト、の順で足すと迷いが少ない
- 技術特化の本（JS, Ruby, React 等）はバージョン依存が強い。**刊行年と対応バージョン**を確認してから提案する
