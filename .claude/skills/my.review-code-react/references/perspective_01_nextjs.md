このファイルは仕様書ではなく、レビュー時に見落としを防ぐための観点メモです。
各項目で「何を疑うべきか」を短く確認し、仕様断定が必要になったら `next-docs/` か公式 docs に進みます。

## ルーティング

- ルーターの方針が統一されているか
- 動的ルートや並列ルートが必要以上に複雑でないか
- Route Handler が UI 層と混ざっていないか

## レンダリング戦略

- Server と Client の境界が妥当か
- `use client` を広げすぎていないか
- SSR / SSG / CSR の選択に理由があるか

## データ取得

- Server で取るべきデータを Client で取っていないか
- mutation 後の再取得や再検証が整理されているか
- loading / error state が抜けていないか

## キャッシュ

- `cache`, `revalidate`, `no-store` が意図的に設定されているか
- stale データや過剰再取得のリスクがないか

## SEO / metadata

- 公開ページに metadata があるか
- 必要なら OG, sitemap, robots を扱っているか

## 型安全性

- `any` や強引な型アサーションが多くないか
- route params や API 応答の型が崩れていないか

## 環境変数

- `NEXT_PUBLIC_` を付けるべき値と付けてはいけない値が分かれているか
- 機密値が Client に漏れていないか

## エラーハンドリング

- 失敗時の UI と回復導線があるか
- 404 と想定外エラーが分けられているか

## 画像 / アセット

- 画像最適化と LCP への配慮があるか
- 大きいアセットを無自覚に配信していないか

## 国際化

- 文言、日時、数値が locale 前提で壊れないか
- i18n が不要なプロダクトでも拡張不能な実装になっていないか
