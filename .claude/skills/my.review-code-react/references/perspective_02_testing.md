## Vitest / RTL チェック

## 目次

- [Vitest / RTL チェック](#vitest--rtl-チェック)
- [典型的な問題](#典型的な問題)

テストレビューでは、書き方の綺麗さより「壊れるときに本当に検知できるか」を見ます。
このファイルは flaky さと実装依存を見抜くための短い確認表です。

- 重要な成功系、失敗系、境界条件を押さえているか
- DOM 構造ではなくユーザーの振る舞いを検証しているか
- `screen` と `userEvent` を優先し、querySelector 依存になっていないか
- 非同期テストで待機方法が適切か
- モックが多すぎて実装差分を見逃さないか
- テスト名だけで何を守っているか分かるか

## 典型的な問題

- スナップショットだけで本質的な振る舞いを確認しない
- `setTimeout` や日時依存を固定せず flaky になる
- 実装詳細に寄ったモックで安心してしまう
